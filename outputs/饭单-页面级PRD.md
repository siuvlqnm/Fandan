# 饭单 页面级 PRD

## 1. 文档目的

本文档基于《饭单 MVP 第一版方案》拆解第一版页面级产品需求，用于后续 UI 设计、前端开发、后端接口设计和验收。

第一版目标是跑通核心闭环：

> 新建饭单 -> 添加菜品 -> 分享确认 -> 生成购物清单 -> 存档复用

## 2. 产品角色

### 创建者

登录后使用饭单的人，可以是家庭成员、做饭阿姨、私厨、聚餐组织者。

权限：

- 创建和管理用餐对象
- 创建和编辑菜品
- 创建和编辑饭单
- 生成购物清单
- 生成分享确认链接
- 查看访客反馈
- 归档和复用饭单

### 访客

通过分享链接打开饭单的人，可以是客户、家人、朋友、雇主。

权限：

- 查看饭单
- 对菜品反馈
- 提交忌口备注
- 确认饭单

第一版访客不需要注册。

## 3. 页面总览

第一版包含以下页面：

1. 登录页
2. 首页
3. 饭单列表页
4. 新建/编辑饭单页
5. 饭单详情页
6. 菜品列表页
7. 新建/编辑菜品页
8. 用餐对象列表页
9. 新建/编辑用餐对象页
10. 购物清单页
11. 分享确认页
12. 设置页

## 4. 登录页

### 目标

让创建者进入自己的工作空间。

### 第一版能力

- 邮箱登录
- 验证码或 magic link 登录
- 登录后进入首页

### 可后置能力

- 手机号登录
- 微信登录
- 多账号绑定

### 页面元素

- 产品名：饭单
- 定位文案：给家人和客户安排菜单、确认忌口、生成购物清单
- 邮箱输入框
- 获取验证码/发送登录链接按钮
- 登录说明

### 验收标准

- 用户输入邮箱后可以发起登录
- 登录成功后进入首页
- 未登录访问创建者页面时跳转登录页
- 分享确认页不要求登录

## 5. 首页

### 目标

让用户快速继续最近的饭单或创建新饭单。

### 页面模块

- 顶部快捷操作：新建饭单、新建菜品、新建对象
- 今日/本周饭单
- 待确认饭单
- 最近用餐对象
- 最近菜品
- 最近购物清单

### 空状态

新用户没有数据时展示：

- 主要按钮：新建第一份饭单
- 次要按钮：先添加一个用餐对象

### 验收标准

- 用户可从首页进入新建饭单流程
- 有待确认饭单时优先展示
- 点击饭单卡片进入饭单详情
- 移动端首屏能看到核心操作按钮

## 6. 饭单列表页

### 目标

管理所有饭单，支持查找、筛选和复用。

### 页面元素

- 新建饭单按钮
- 搜索框
- 状态筛选：全部、草稿、待确认、已确认、已完成、已归档
- 类型筛选：全部、单餐、一日、一周、聚餐
- 饭单列表

### 饭单卡片字段

- 标题
- 用餐对象
- 日期或日期范围
- 类型
- 状态
- 菜品数量
- 最近更新时间

### 卡片操作

- 查看详情
- 复制
- 归档

### 验收标准

- 可按状态筛选饭单
- 可搜索饭单标题和对象名称
- 可复制历史饭单生成新饭单
- 归档饭单默认不出现在活跃列表中

## 7. 新建/编辑饭单页

### 目标

让用户用尽量少的步骤创建饭单。

### 新建流程

1. 输入饭单标题
2. 选择饭单类型
3. 选择或新建用餐对象
4. 设置日期或日期范围
5. 选择餐别
6. 保存并进入饭单详情

### 饭单类型

- 单餐
- 一日
- 一周
- 聚餐

### 快捷创建

如果没有用餐对象，允许在当前页面快速创建：

- 对象名称
- 类型
- 人数
- 忌口/口味备注

### 验收标准

- 没有菜品库时也可以创建饭单
- 创建后进入饭单详情
- 一周饭单支持日期范围
- 聚餐饭单支持时间和人数备注

## 8. 饭单详情页

### 目标

饭单详情页是第一版的核心工作台。

### 页面区域

#### 顶部信息

- 饭单标题
- 用餐对象
- 日期
- 状态
- 分享按钮
- 更多操作：复制、归档、删除

#### 对象提示

- 人数
- 口味偏好
- 忌口/过敏
- 预算备注

#### 菜品区域

根据饭单类型展示：

- 单餐：按菜品列表展示
- 一日：按早餐、午餐、晚餐、加餐展示
- 一周：按日期 + 餐别展示
- 聚餐：按凉菜、热菜、主食、汤、饮品等分组，可第一版简化为列表

#### 菜品操作

- 添加已有菜品
- 快速新建菜品
- 移除菜品
- 调整顺序
- 添加备注

#### 反馈区域

- 显示访客反馈
- 按菜品聚合喜欢、不喜欢、想替换
- 展示忌口备注
- 展示确认状态

#### 底部操作

- 生成/查看购物清单
- 生成分享链接
- 标记为已确认
- 标记为已完成
- 归档

### 状态流转

- 草稿：创建者编辑中
- 待确认：已生成分享链接，等待反馈
- 已确认：客户或创建者确认
- 已完成：采购/做饭完成
- 已归档：进入历史

### 验收标准

- 可在详情页完成添加菜品、生成购物清单、分享确认
- 添加菜品后饭单自动保存
- 访客反馈提交后创建者能在详情页看到
- 已归档饭单不可直接编辑，需要复制后编辑

## 9. 菜品列表页

### 目标

管理用户自己的菜品库，并支持快速复用。

### 页面元素

- 新建菜品按钮
- 搜索框
- 分类筛选
- 标签筛选
- 菜品列表

### 菜品卡片字段

- 菜名
- 分类
- 主要食材
- 标签
- 最近使用时间

### 操作

- 查看/编辑
- 添加到饭单
- 删除

### 验收标准

- 可搜索菜名、食材、标签
- 可按分类筛选
- 可从菜品列表添加到某个饭单

## 10. 新建/编辑菜品页

### 目标

让用户低成本录入可复用菜品。

### 字段

- 菜名，必填
- 分类
- 描述
- 食材列表
- 简单做法
- 标签
- 可见范围

### 食材字段

- 名称
- 数量
- 单位
- 分类
- 备注

第一版数量和单位均可选，允许只填食材名称。

### 验收标准

- 只填菜名也可以保存
- 食材为空时生成购物清单时跳过该菜
- 新建菜品后可以立即添加到饭单

## 11. 用餐对象列表页

### 目标

管理家庭、客户、聚餐对象等。

### 页面元素

- 新建对象按钮
- 搜索框
- 类型筛选
- 对象列表

### 对象卡片字段

- 名称
- 类型
- 人数
- 忌口摘要
- 最近饭单
- 最近更新时间

### 操作

- 查看/编辑
- 查看历史饭单
- 新建该对象的饭单

### 验收标准

- 可按对象名称搜索
- 可按类型筛选
- 可从对象直接新建饭单

## 12. 新建/编辑用餐对象页

### 目标

记录菜单协作需要用到的基础信息。

### 字段

- 名称，必填
- 类型：我家、客户、聚餐、其他
- 人数
- 口味偏好
- 忌口/过敏
- 预算备注
- 联系备注

### 验收标准

- 只填名称也可以保存
- 创建饭单时可以快速新建对象
- 编辑对象后，新饭单使用最新对象信息
- 历史饭单保留创建时的备注快照可后置

## 13. 购物清单页

### 目标

让用户基于饭单完成采购准备。

### 生成规则

第一版规则保持简单：

- 从饭单中所有菜品读取食材
- 按食材名称聚合
- 数量和单位相同则合并
- 数量或单位不一致则保留多条，允许用户手动整理
- 没有分类的食材归为其他

### 页面元素

- 所属饭单
- 重新生成按钮
- 添加物品按钮
- 分类分组
- 购物项列表

### 购物项字段

- 名称
- 数量
- 单位
- 分类
- 是否已购买
- 备注

### 操作

- 勾选已购买
- 编辑
- 删除
- 添加自定义项

### 验收标准

- 饭单有菜品和食材时可以生成购物清单
- 用户手动修改后不会被自动覆盖，除非主动点击重新生成
- 勾选状态可保存
- 移动端买菜时可单手勾选

## 14. 分享确认页

### 目标

让客户、家人或参与者无需登录即可确认饭单。

### 页面内容

- 饭单标题
- 日期和用餐对象
- 创建者备注
- 菜品列表
- 反馈入口
- 忌口备注输入
- 确认按钮

### 菜品反馈

每道菜支持：

- 喜欢
- 不喜欢
- 想替换
- 备注

### 全局反馈

- 我有忌口
- 我想补充
- 我确认这份饭单

### 验收标准

- 访客打开链接无需登录
- 访客提交反馈后创建者可见
- 同一访客可以修改自己的反馈可后置
- 分享链接过期后不可访问

## 15. 设置页

### 第一版内容

- 个人资料
- 工作空间名称
- 默认饭单类型
- 默认菜品分类
- 退出登录

### 后置内容

- 订阅管理
- 品牌分享页设置
- 团队成员
- 数据导出

## 16. 第一版验收清单

第一版上线前必须满足：

- 用户可以登录
- 用户可以创建用餐对象
- 用户可以创建菜品
- 用户可以创建饭单
- 用户可以把菜品加入饭单
- 用户可以生成购物清单
- 用户可以生成分享链接
- 访客可以查看饭单并反馈
- 创建者可以查看反馈
- 用户可以确认、完成、归档饭单
- 用户可以复制历史饭单
- 移动端核心流程可用

## 17. 重要产品原则

- 第一次使用先让用户做出饭单，不强迫维护完整资料
- 饭单是核心，不是菜品库
- 分享页要极简，访客不登录
- 第一版购物清单可编辑即可，不追求智能单位换算
- 第一版不做内容社区、不做交易撮合、不做医疗营养建议
